# 竞品页面捕捉器

一个面向竞品研究的 Figma 插件方案：在插件里输入 URL，先做可访问性校验，再通过本地 Playwright 抓取服务自动分析页面交互点、截取对应状态或跳转页截图，最后把选中的截图导入 Figma 画布。

## 架构

- `code.ts`: Figma 主线程，负责展示插件 UI、接收导入消息、将图片写入当前画布。
- `ui.html`: 插件界面。当前唯一生效的 UI 来源，负责 URL 校验、任务轮询、截图列表展示、选择与导入。
- `service/`: 本地抓取服务。负责调用 Playwright 校验 URL、识别交互点、点击并截图、暴露本地 HTTP API 给插件 UI。

之所以采用“插件 + 本地服务”的组合，而不是把爬虫直接塞进 Figma 插件，是因为 Playwright 需要完整的 Node.js 运行时和本地浏览器控制能力，而 Figma 插件沙箱更适合 UI 与导入逻辑。

## 技术选型

- 截图与交互抓取：`playwright-core`
- Figma 导入：`figma.showUI`、`figma.ui.onmessage`、`figma.createImage`
- UI：原生 HTML/CSS/JS，直接内联在 `ui.html` 中，避免额外打包层，便于后续维护

## 本地运行

1. 构建主线程代码：

   ```bash
   npm run build
   ```

2. 启动本地抓取服务：

   ```bash
   npm run service
   ```

3. 在 Figma Desktop 中导入本目录的 `manifest.json`。
4. 运行插件后输入 URL，等待自动校验成功，再点击“获取截图”。
5. 勾选想导入的截图，点击“导入”即可写入当前画布。

## 使用说明

- URL 输入框会自动触发校验。校验失败会直接展示中文原因。
- “获取截图”会抓取：
  - 原始落地页
  - 当前页面上检测到的可交互元素
  - 每个交互点点击后的页面状态
  - 因点击打开的新页面或跳转后的页面截图
- “停止”会请求本地服务终止当前任务。
- “导入”只会导入已勾选且有可用视觉结果的截图。

## 重要说明

- 插件 UI 默认连接 `http://localhost:3845`。
- 当前实现会优先分析初始页面上检测到的主要可交互元素，并截图它们触发后的结果页/状态页。
- 为了绕过 Figma 单张图片 4096px 的限制，长页面会被自动切成多段 PNG 后再导入。

## 后端部署

`service/` 已经可以作为独立后端部署，包含任务队列、并发限制、产物清理、基础限流、可选 API Key 和内网地址防护。部署到 Figma Community 可用的线上服务前，请阅读 [`DEPLOYMENT.md`](./DEPLOYMENT.md)。
